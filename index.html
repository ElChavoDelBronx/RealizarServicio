Controller

El Controller es la puerta de entrada de tu aplicación. Recibe las peticiones web (HTTP como GET, POST) , las mapea a funciones específicas, extrae los datos necesarios y delega la tarea a la capa de Service (lógica de negocio). Finalmente, recibe la respuesta del Service, la formatea y la envía de vuelta al cliente con el código de estado HTTP apropiado (200 OK, 201 Created, etc.).

Dtos

El DTO (Data Transfer Object) es un objeto plano que actúa como un contrato de datos o mensajero. Se usa para transferir solo la información necesaria entre las capas (Controller, Service) y el cliente, lo que garantiza la seguridad al no exponer la entidad completa (Model) y facilita el desacoplamiento de tu aplicación.

Ejemplo

Model

El Model (Entidad) es la representación de una tabla en la base de datos. Es la clase que define la estructura de los datos y su estado persistente, sirviendo como la fuente de verdad de la información dentro de la capa de persistencia (Repository).

Ejemplo

Repository

El Repository es la capa de persistencia responsable de comunicarse con la base de datos. Sirve como un contrato para las operaciones CRUD y permite consultas personalizadas utilizando el Model (Entidad).

Ejemplo

Service

El Service es la capa de lógica de negocio de la aplicación. Ejecuta todas las reglas y validaciones del sistema, siendo llamado por el Controller y coordinando las interacciones con el Repository para asegurar que las operaciones se realicen correctamente.
